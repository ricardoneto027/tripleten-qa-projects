# Sprint 2 — Design de Testes: Urban Routes

Projeto focado em design de testes para a funcionalidade de **partilha de carros** do Urban Routes, com base em requisitos fornecidos. O trabalho centrou-se no formulário "Adicionar carteira de motorista".

---

## 🗺️ Contexto

| Campo | Detalhe |
|---|---|
| Aplicação | Urban Routes — funcionalidade de partilha de carros |
| Tipo de trabalho | Design de testes |
| Técnica utilizada | Classes de equivalência e valores limite |
| Ferramenta | Google Sheets |
| Base | Requisitos oficiais do Urban Routes |

---

## 📁 Ficheiros

| Ficheiro | Descrição |
|---|---|
| `test-cases.md` | Casos de teste desenhados com base nas classes de equivalência |
| `classes-de-equivalencia.md` | Classes de equivalência para otimização de testes |

---

## 🔍 O que foi feito

- Análise dos requisitos do formulário "Adicionar carteira de motorista"
- Identificação de classes de equivalência (válidas e inválidas)
- Desenho de casos de teste com base nas classes identificadas
- Cobertura de campos: nome, apelido, número da carta de condução, validade

---

## 🧠 Técnicas aplicadas

| Técnica | Descrição |
|---|---|
| Classes de equivalência | Agrupamento de inputs com comportamento esperado igual |
| Valores limite | Testes nos extremos de cada intervalo válido |
